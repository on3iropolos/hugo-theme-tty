# The Case for Plain Text

*April 10, 2026*

## The Formats We Use

Word processors, rich text editors, Notion, Evernote. They all save to proprietary formats. Your notes are trapped in their ecosystems.

## Why Plain Text Wins

### Portability
Plain text opens everywhere. TextEdit, Vim, VS Code, cat. No software required. No format migrations.

### Version Control
Diff plain text. Git loves it. Merge conflicts are resolvable. Track every change.

### Tool Freedom
Choose your editor. Choose your workflow. Plain text is the lowest common denominator.

### Longevity
Will .docx files open in 20 years? Maybe. Will .txt files? Definitely. Plain text is future-proof.

### Speed
No formatting menus. No style panels. Just write. Markdown adds just enough structure without the overhead.

## The Ecosystem

- **Markdown** - Lightweight markup for documentation and notes
- **Org-mode** - Plain text organization for Emacs users
- **YAML/TOML** - Configuration that humans can read
- **JSON** - Data interchange, though sometimes overused

## The Tradeoffs

Plain text lacks WYSIWYG. It lacks collaborative editing. It lacks some features. But it provides something more valuable: ownership of your data.

## Make the Switch

Start with one plain text file. Convert your notes. Write your next document in Markdown. Notice the difference.

Plain text isn't primitive. It's essential.
